# 工作管理仓库协作约定

本目录是工作管理器和 Codex 共同使用的本地工作空间。

## 处理项目需求前

1. 从需求中识别项目 ID。
2. 先读取 `project-rules/<项目 ID>.md`。
3. 再读取 `projects/<项目 ID>.yaml`，确认仓库、默认分支与服务配置。
4. 将需求、决策和进展沉淀到对应任务的 `data/artifacts/<任务 ID>/`。

## 风险操作

未读取项目规则，或规则不存在时，先说明缺失并获得确认；不得直接执行会影响项目的 Git、合并、发布、远程 Issue、外部服务或数据删除操作。

Demo 项目只用于展示，不连接真实代码仓库。不得尝试为它创建 worktree、分支、Issue 或开发服务。
