# 桌面端布局精简设计

## 目标

将工作管理器桌面端收敛为左侧导航与右侧内容区，使项目与任务页面更快进入核心操作。

## 范围

- 应用壳改为固定左侧导航，保留看板、项目、设置三个入口，并为 macOS 窗口控制按钮预留顶部空间。
- 项目页顶部只保留项目搜索和同步按钮，项目卡片继续打开项目详情。
- 任务页移除页面标题、说明和统计，只保留搜索、项目、状态、优先级筛选和状态列看板。
- 保留现有 wm API、同步、筛选、任务详情与项目详情行为。

## 交互与可访问性

- 当前导航项以文字、图标和浅色背景共同表达，导航按钮保留可访问名称。
- 搜索字段使用显式标签，支持按项目名称、ID 和默认分支过滤。
- 任务筛选保持现有 `aria-label` 和键盘可操作的原生选择框。
- 加载、空状态、同步禁用态和减少动态效果的偏好保持原有行为。

## 视觉约束

- 沿用现有氧化青色令牌和 8 至 12px 圆角，不引入新依赖或装饰性阴影。
- 侧栏固定宽度为 112px，内容区在宽屏使用剩余空间；窄屏时侧栏转换为顶部横向入口。
- 项目卡片在内容区采用自适应网格；看板在宽屏显示四列，中小屏按既有断点折行。

## 验证

- 桌面端测试覆盖左侧导航、项目搜索、同步与看板筛选。
- 运行 `pnpm --filter @work-manager/desktop test --run`、`pnpm --filter @work-manager/desktop typecheck` 和 `git diff --check`。
