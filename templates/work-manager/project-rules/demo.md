## 项目定位

- 项目 ID：`demo`
- 代码仓库：无。该项目只展示工作管理器的数据流。
- 默认分支：`main`（仅为示例）。

## 分支约定

- 真实项目可使用 `feature/<任务号>-<简述>`、`fix/<任务号>-<简述>` 等命名。
- Demo 不创建分支或 worktree。

## 合并约定

- 真实项目应先完成项目定义的测试、类型检查和代码评审，再合并到默认分支。
- Demo 不存在 PR 或合并流程。

## 部署方式

- 真实项目应写明环境、执行命令、回滚步骤和负责人。
- Demo 不部署、不启动服务，也不访问外部环境。

## 禁止事项

- 不把 Demo 的占位路径当作真实代码仓库。
- 不为 Demo 创建 Git 分支、worktree、Issue 或开发服务。
