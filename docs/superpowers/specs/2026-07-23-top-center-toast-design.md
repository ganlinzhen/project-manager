# 顶部居中 Toast 设计

## 目标

将桌面端的页面内操作反馈替换为不占用内容区域的顶部居中 Toast，降低连续操作时的干扰。

## 范围

- 成功与失败反馈统一通过 `App` 的通知队列展示。
- Toast 固定在顶部居中，默认展示 5 秒。
- 鼠标悬停暂停倒计时；用户可以手动关闭。
- 最多同时显示 3 条，后续消息排队；错误仍使用 `role="alert"`，成功使用 `role="status"`。

## 架构

新增纯展示组件 `ToastViewport`，输入为 `Toast[]` 和关闭回调。`App` 维护队列并提供 `pushToast`：现有的 `setFeedback` 调用替换为此入口，任务详情页不再在正文中嵌入反馈组件。每个 Toast 在组件内部管理自己的 5 秒计时与悬停暂停，不引入第三方库。

## 边界

- 不改变 wm API、任务操作或 Tauri 原生逻辑。
- 不做持久化、不支持跨会话恢复。
- 当同一时刻超过 3 条通知，队列按创建顺序依次晋升，不丢弃通知。

## 验收

1. 操作反馈显示在顶栏下方、页面水平居中，不改变正文布局。
2. 成功和失败均在 5 秒后消失，悬停期间不消失，可手动关闭。
3. 连续通知最多显示 3 条，关闭/超时后下一个自动出现。
4. 现有桌面端测试和新增 Toast 行为测试通过。
