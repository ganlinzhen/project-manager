# 工作管理仓库协作约定

本目录是工作管理器和 Codex 共同使用的本地工作空间。

## 处理项目需求前

1. 从需求中识别项目 ID。
2. 先读取 `project-rules/<项目 ID>.md`。
3. 再读取 `projects/<项目 ID>.yaml`，确认仓库、默认分支与服务配置。
4. 需求清晰时直接实施；不强制执行“设计、计划、实施”的 spec 流程。

## 任务执行与记录

- 本仓库的日常任务不生成 spec、设计文档、实施计划或任务工件；不要在 `docs/`、`claude_doc/`、`data/artifacts/` 或其他目录创建此类文档。
- 每次实际修改后，只在仓库根目录 `change.log` 追加一条记录，包含操作时间、修改内容和验证结果。
- 用户明确要求设计文档、计划或其他交付文档时，才生成该文档；即使生成，仍需在 `change.log` 留痕。
- 不因等待设计确认或计划评审而阻塞范围清晰的实现；风险操作仍遵守下方规则。

## 风险操作

未读取项目规则，或规则不存在时，先说明缺失并获得确认；不得直接执行会影响项目的 Git、合并、发布、远程 Issue、外部服务或数据删除操作。

Demo 项目只用于展示，不连接真实代码仓库。不得尝试为它创建 worktree、分支、Issue 或开发服务。
